Programowanie obiektowe w Pythonie

Definicje • Przykłady • Dobre praktyki • (ABC, iteratory, wzorce)

1. Wszystko jest obiektem

W Pythonie każda wartość to obiekt (ma typ, atrybuty i metody).

Obiekt = stan (atrybuty) + zachowanie (metody).

Przykład


x = 5
print(type(x))   # 

s = "hello"
print(type(s))   #

Liczby, napisy, listy, funkcje, klasy — wszystko to obiekty.

2. Klasy i obiekty

Klasa — definicja typu (jak ma wyglądać i działać obiekt).

Obiekt — konkretny egzemplarz klasy (ze swoim stanem).

Definicja i tworzenie instancji

  • __init__ — konstruktor; przygotowuje świeży obiekt.
  • self — bieżąca instancja (jak this w innych językach).
  • Atrybuty przypisujemy jako self.nazwa = wartość.

class Samochod:
    def __init__(self, marka, model):
        self.marka = marka
        self.model = model

auto = Samochod("Toyota", "Camry")
print(auto.marka)  # Toyota

3. Atrybuty i metody

Atrybut — dane przechowywane w obiekcie.

Metoda — funkcja wywoływana na obiekcie (operuje na jego stanie).

  • Metody mają self jako pierwszy argument.
  • Modyfikują stan przez self.coś = ....
  • Zwracają wynik (opcjonalnie).

class Kalkulator:
    def __init__(self):
        self.wynik = 0

    def dodaj(self, x):
        self.wynik += x
        return self.wynik

k = Kalkulator()
print(k.dodaj(3))  # 3

4. Dziedziczenie

Klasa potomna przejmuje kod klasy bazowej i może go rozszerzać lub nadpisywać.

Ułatwia ponowne użycie kodu i modelowanie hierarchii typów.

Podstawy


class Pojazd:
    def ruszaj(self):
        print("Pojazd rusza.")

class Samochod(Pojazd):
    def zatrzymaj(self):
        print("Samochód zatrzymuje się.")

auto = Samochod()
auto.ruszaj()
auto.zatrzymaj()

super()

  • super() wywołuje implementację z klasy bazowej.
  • Przydatne w konstruktorach i metodach nadpisanych.

class Zwierze:
    def __init__(self, imie):
        self.imie = imie

class Pies(Zwierze):
    def __init__(self, imie, rasa):
        super().__init__(imie)
        self.rasa = rasa

Dziedziczenie wielokrotne


class Pierwsza:
    def metoda1(self): print("z Pierwsza")

class Druga:
    def metoda2(self): print("z Druga")

class Potomna(Pierwsza, Druga):
    pass

obj = Potomna()
obj.metoda1()
obj.metoda2()

Python używa MRO (Method Resolution Order), żeby ustalić kolejność wyszukiwania metod.

Przesłanianie metod (override)


class Kolo:
    def info(self):
        print("To jest kółko.")

class KoloKolorowe(Kolo):
    def info(self):
        print("To jest kolorowe kółko.")

5. Polimorfizm

Wspólny interfejs, różne implementacje.

Kod kliencki nie musi znać konkretnego typu obiektu.

Operator overloading


class Punkt:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __add__(self, other):
        return Punkt(self.x + other.x, self.y + other.y)

a = Punkt(1,2)
b = Punkt(3,4)
c = a + b

Obiekt wywoływalny


class Licznik:
    def __init__(self):
        self.liczba = 0
    def __call__(self):
        self.liczba += 1
        return self.liczba

licznik = Licznik()
print(licznik()) # 1
print(licznik()) # 2

Polimorfizm „w praktyce”


class Zwierze:
    def mowa(self): raise NotImplementedError

class Pies(Zwierze):
    def mowa(self): return "hau"

class Kot(Zwierze):
    def mowa(self): return "miau"

def powiedz_cos(z: Zwierze):
    print(z.mowa())  # działa dla różnych typów

powiedz_cos(Pies()); powiedz_cos(Kot())

6. Hermetyzacja

Ukrywanie szczegółów implementacji; kontrolowany dostęp do stanu.

Atrybuty: public/protected/private*

  • name — „publiczny” (konwencja).
  • _name — „chroniony” (konwencja).
  • __name — prywatny (name mangling).

class Konto:
    def __init__(self, saldo):
        self.saldo = saldo     # public (konwencja)
        self._limit = 1000     # protected (konwencja)
        self.__pin = "1234"    # private (mangling)

print(dir(Konto(10)))  # zobacz _Konto__pin

@property — bezpieczny dostęp


class Osoba:
    def __init__(self, imie, nazwisko):
        self._imie = imie
        self._nazwisko = nazwisko

    @property
    def pelne_nazwisko(self):
        return f"{self._imie} {self._nazwisko}"

7. Abstrakcja (ABC)

Klasy abstrakcyjne (moduł abc) definiują interfejs bez implementacji.

Klasy pochodne muszą dostarczyć implementacje metod abstrakcyjnych.


from abc import ABC, abstractmethod

class Figura(ABC):
    @abstractmethod
    def pole(self): ...

class Kolo(Figura):
    def __init__(self, r): self.r = r
    def pole(self): return 3.14159 * self.r**2

class Prostokat(Figura):
    def __init__(self, a, b): self.a, self.b = a, b
    def pole(self): return self.a * self.b

8. Iteratory i generatory

„Jak przechodzić po danych” — kluczowe w Pythonie.

Iterator (prosty)


class Licznik:
    def __init__(self, n): self.n = n
    def __iter__(self): return self
    def __next__(self):
        if self.n <= 0: raise StopIteration
        self.n -= 1
        return self.n

Generator


def licznik(n):
    while n > 0:
        n -= 1
        yield n

for x in licznik(3):
    print(x)

9. Wyjątki a OOP

Błędy są obiektami; tworzymy własne typy wyjątków.

Własny wyjątek domenowy


class BladSaldo(Exception): pass

class Konto:
    def __init__(self, saldo): self.saldo = saldo
    def wyplac(self, kwota):
        if kwota > self.saldo:
            raise BladSaldo("Za mało środków!")
        self.saldo -= kwota

10. Wzorzec: Singleton (Python)

Jedna instancja w całej aplikacji — ostrożnie i świadomie.

Minimalna implementacja


class Singleton:
    _inst = None
    def __new__(cls, *a, **k):
        if cls._inst is None:
            cls._inst = super().__new__(cls)
        return cls._inst

a = Singleton()
b = Singleton()
print(a is b)  # True

Podsumowanie

  • W Pythonie wszystko jest obiektem.
  • Klasy łączą dane i zachowania; hermetyzuj stan.
  • Dziedziczenie + polimorfizm = elastyczny i rozszerzalny kod.
  • ABC, iteratory/generatory i wzorce porządkują projekt.
